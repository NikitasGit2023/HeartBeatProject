# מערכת ניטור HeartBeat

אפליקציית .NET 8 לניטור בריאות מערכות באמצעות מנגנון "דופק" מבוסס קבצים. המערכת מזהה כשתהליך מרוחק מפסיק לפעול ושולחת התרעות במייל.

---

## סקירה כללית

תהליכים רבים רצים ברקע ללא כל משוב — עבודות אוטומטיות, סוכנים מרוחקים, שירותי רקע. אם הם קורסים, אף אחד לא יודע על כך עד שמשהו אחר נשבר.

HeartBeat פותרת את זה: התהליך המנוטר כותב קובץ קטן במרווחי זמן קבועים. מנגנון נפרד בודק שהקובץ מתעדכן — אם הוא נעלם או ישן מדי, נשלחת התרעה.

---

## ארכיטקטורה

המערכת מורכבת משני תפקידים:

| תפקיד | שם | תיאור |
|--------|----|--------|
| **TX** | משדר | כותב קובץ דופק עם חותמת זמן במרווחים קבועים |
| **RX** | מקלט | מנטר תיקייה; מתריע אם הקובץ נעלם או ישן מדי |

בפריסה אמיתית, TX ו-RX רצים על **מכונות נפרדות** עם גישה לתיקייה משותפת ברשת. בפרויקט זה, שניהם **מדומים על אותה מכונה** באמצעות שתי תיקיות מקומיות שונות.

---

## יכולות

- **יצירת דופק** — TX כותב קובץ `.txt` עם חותמת זמן במרווח הניתן להגדרה
- **ניטור סף** — RX מסמן את המערכת כ-DOWN אם הקובץ האחרון ישן מדי
- **התרעות במייל** — שליחת מייל במעבר HEALTHY → DOWN, עם מנגנון קולדאון של 5 דקות למניעת הצפה
- **לוגים חיים** — כל אירועי השירות נלכדים בלוג זיכרון מקוטגר (TX / RX / Email / System)
- **ממשק ווב** — לוח בקרה בדפדפן המציג סטטוס, הגדרות ולוג חי
- **מתקין** — אשף WPF שמגדיר ומתקין את המערכת כ-Windows Service

---

## התקנה

### 1. פרסום השרת

```bash
dotnet publish HeartBeatProject.server/HeartBeatProject.server.csproj -c Release -r win-x64 --self-contained true
```

### 2. הפעלת המתקין

הפעל את `HeartBeatProjectInstaller.exe` כמנהל ועקוב אחרי האשף:

1. **ברוכים הבאים** — מבוא
2. **הגדרות** — בחר מצב TX או RX, הגדר נתיבי תיקיות, מרווחי זמן ופרטי SMTP אופציונליים
3. **התקנה** — האשף כותב את `appsettings.json` ורושם את ה-Windows Service
4. **סיום** — השירות מתחיל אוטומטית

---

## שימוש

לאחר ההתקנה, השירות רץ ברקע כ-Windows Service ומופעל אוטומטית עם ההפעלה.

לגישה לממשק הווב, פתח דפדפן ונווט לכתובת:

```
http://localhost:5000
```

לוח הבקרה מציג:
- סטטוס נוכחי (HEALTHY / DOWN)
- מצב פעיל (TX / RX)
- זמן הדופק האחרון
- לוג חי עם סינון לפי קטגוריה (TX, RX, Email, System)

ניתן לשנות הגדרות בזמן ריצה מדף ההגדרות ללא הפעלה מחדש של השירות. שים לב שהשינויים אינם נשמרים לדיסק — לשינויים קבועים, ערוך את `appsettings.json` והפעל מחדש את השירות.

---

## הגדרות

השירות מוגדר באמצעות `appsettings.json`:

```json
{
  "Heartbeat": {
    "Mode": "TX",
    "FolderPath": "C:\\HeartbeatFiles",
    "IntervalSeconds": 30,
    "CheckIntervalSeconds": 10,
    "ThresholdSeconds": 60
  },
  "Alerts": {
    "EnableEmail": true,
    "SmtpServer": "smtp.example.com",
    "Port": 587,
    "From": "monitor@example.com",
    "To": "admin@example.com",
    "Username": "",
    "Password": "",
    "EnableSsl": true
  }
}
```

---

## הערות

- TX ו-RX מיועדים למכונות נפרדות אך מדומים באותה מכונה בפרויקט זה
- התרעות מייל דורשות פרטי SMTP מלאים — אם חסר פרט כלשהו, ההתרעה נדחית ונרשמת אזהרה בלוג
- ניתן להוסיף ספקי התרעות נוספים (Syslog, SNMP) על ידי מימוש הממשק `IAlertService`
